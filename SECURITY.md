# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.0.0     | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability in YULA AI Scanner, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

### How to report

Send a description of the vulnerability to the author:

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested mitigations

### What to expect

- Acknowledgement within 48 hours
- A fix or mitigation plan within 14 days for critical issues
- Credit in the changelog/release notes (if desired)

## Scope

YULA AI Scanner is a security testing tool. Please note:
- Vulnerabilities in YULA AI Scanner itself (the CLI/Python package) are in scope
- Vulnerabilities in AI systems that YULA AI Scanner tests are out of scope for this policy
- Issues with third-party dependencies should be reported upstream

## Safe Harbour

We will not pursue legal action against researchers who:
- Report vulnerabilities in good faith via the channel above
- Do not access, modify, or exfiltrate production data
- Do not disrupt services
